# Audio Pipeline (opt-in) — Poalim Presentation

The narration audio for the speaker-notes companion is generated by ElevenLabs TTS. **This is opt-in.** Run it only when the user explicitly asks ("generate the narration", "produce the audio", "run TTS", "make the speaker audio").

## Prerequisites

- `ELEVENLABS_API_KEY` environment variable. The user must provide it; never embed a key in code.
- `ffmpeg` installed locally (already present on the user's machine).
- Python 3 with the `elevenlabs` package: `pip install elevenlabs`.
- A populated `speaker-notes.html` (or whichever slug the user is using) with `<p>` / `<li>` narration inside `<div class="section">` blocks.

## The script

The canonical generator is `scripts/generate-audio.py` in the project root. Read it before invoking. Key facts:

- **Model: `eleven_v3`** — this is the only ElevenLabs model that produces Hebrew. `eleven_multilingual_v2` does NOT produce Hebrew (verified the hard way; documented in CLAUDE.md).
- **Voice: `cgSgspJ2msm6clMCkdW9`** — the "Jessica" preset.
- **Voice settings:** `VoiceSettings(stability=0.3, similarity_boost=0.7, style=0.4, speed=1.15, use_speaker_boost=True)`.
- **Hash-cached** including voice id, model, and settings. Changing any of those auto-invalidates the cache and re-renders. Otherwise, only slides whose narration changed are re-rendered.

The parser regex is `<(p|li)(\s[^>]*)?>(.*?)</\1>`. **Do not** loosen this to `<(p|li)([^>]*)>` — that matches `<path>` SVG tags too and leaks SVG content into the narration. Stage-direction paragraphs styled with `style="opacity:0.6"` are intentionally skipped by the script.

## Running the pipeline

```bash
# 1. Generate per-slide MP3s from speaker-notes.html
ELEVENLABS_API_KEY=xi-api-... python3 scripts/generate-audio.py

# 2. Concatenate per-slide chunks into one narration file
cd audio
{ for n in $(seq -f "%02g" 1 N); do echo "file 'slide-${n}.mp3'"; done } > concat-list.txt
ffmpeg -y -f concat -safe 0 -i concat-list.txt -c copy narration.mp3
rm concat-list.txt
```

Replace `N` with the actual slide count.

## What ships in git

- `audio/narration.mp3` — the single concatenated file. **Yes, ship this** — the GitHub Pages deploy needs it.
- `audio/slide-NN.mp3` — per-slide chunks. **No, gitignore these.**
- `audio/*.hash` — cache key files. **No, gitignore these.**

The existing `.gitignore` already covers per-slide chunks and hash files for the example deck. Verify it covers any new audio files you add.

## When the script doesn't exist yet (new project)

If the user is running the audio pipeline in a brand new deck and `scripts/generate-audio.py` doesn't exist yet:

1. Copy the existing script from the example project as the starting point.
2. Update the input HTML path if the companion file is named differently (`<slug>-notes.html` instead of `speaker-notes.html`).
3. Update the slide count in the concat loop.
4. Verify the parser regex is unchanged.

The script structure is roughly:

```python
from elevenlabs.client import ElevenLabs
from elevenlabs import VoiceSettings

client = ElevenLabs(api_key=os.environ["ELEVENLABS_API_KEY"])

VOICE_ID = "cgSgspJ2msm6clMCkdW9"
MODEL = "eleven_v3"
SETTINGS = VoiceSettings(stability=0.3, similarity_boost=0.7, style=0.4, speed=1.15, use_speaker_boost=True)

# Parse <p>/<li> from each <section class="slide"> in speaker-notes.html
# For each slide, hash (text + voice + model + settings); skip if cache hit
# Otherwise call client.text_to_speech.convert(...) and write audio/slide-NN.mp3
# Save the hash to audio/slide-NN.hash
```

## Updating narration

When the user changes narration text in `speaker-notes.html`:

1. Re-run `scripts/generate-audio.py`. The hash cache means only changed slides re-render.
2. Re-run the ffmpeg concat step to produce a fresh `narration.mp3`.
3. Commit `narration.mp3` (only).

If the user changes the voice, model, or settings, every slide will re-render automatically — that's by design.

## When the user asks to change the voice

Common request: "use a male voice" / "more formal" / "slow it down".

- For voice changes, the user provides a new voice ID (or asks you to recommend one). Update `VOICE_ID` in the script.
- For tone changes, adjust `VoiceSettings`:
  - `stability`: 0.3 = expressive (default), 0.6+ = monotone
  - `style`: 0.4 = animated (default), 0.0 = neutral
  - `speed`: 1.0 = baseline, 1.15 = slightly faster (default), 0.9 = slower
- After any change, run the script — it will detect the hash mismatch and re-render everything.

## Privacy note

If the user has a rehearsal recording (e.g., `.m4a` from Zoom) at the project root, that file is **always private** and `.gitignore`-d. Any Whisper transcripts of it are also private. Don't reference them in narration the user wants public.

## QA after audio generation

- Play `narration.mp3` end-to-end and confirm it sounds right.
- Open `speaker-notes.html` in a browser, click play, verify the audio loads.
- Verify navigation between slides does NOT pause or restart the audio (the decoupling is required — see `references/speaker-notes.md`).
- Verify the play button shows elapsed time while playing.
